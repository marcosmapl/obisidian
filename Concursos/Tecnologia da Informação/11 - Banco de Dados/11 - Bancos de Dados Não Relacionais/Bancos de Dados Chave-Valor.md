## Conceito

Modelo mais simples de NoSQL, onde os dados são armazenados como **pares chave → valor**.

A chave funciona como **identificador único**, permitindo acesso rápido ao valor associado.

**Características:**

- Alta performance    
- Estrutura simples
- Fácil escalabilidade

**Limitação:**

- Consulta limitada (normalmente apenas pela chave)

**Exemplos:**

- Redis
- Riak
- Amazon DynamoDB

**Uso comum:**

- Cache
- Sessões de usuário
- Sistemas de alta velocidade
